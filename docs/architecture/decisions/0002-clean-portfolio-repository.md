# ADR 0002: Use a Clean Portfolio Repository

- Status: accepted
- Date: 2026-07-06

## Context

The existing coursework repository contains submitted reports and copies of third-party papers. Its local worktree also records deletions after files were reorganized in the course folder.

## Decision

Create a separate repository for original portfolio code and documentation. Leave the coursework repository and its history untouched. Do not copy lecture decks, assignment pages, Cornell branding assets, or third-party papers into the new repository.

## Consequences

The portfolio has a clear license and professional history. Provenance from the coursework is documented in prose rather than by redistributing source materials.
